# Ice Business Ledger — Spreadsheet Setup

Google Sheets workbook for tracking route sales (cash vs credit per store), store balances, expenses, inventory, and monthly P&L + cash flow.

## Quick start

1. Open [Google Sheets](https://sheets.google.com) → **File → Import**
2. Upload [`templates/ice-business-ledger.xlsx`](../templates/ice-business-ledger.xlsx) (generated by `scripts/build_ledger_workbook.py`)
3. Import type: **Replace spreadsheet** (recommended for a fresh copy)
4. After import, **test dropdowns** — click Route_Sales column B. You should see a dropdown arrow.
5. If dropdowns are missing, see [Dropdown setup after import](#dropdown-setup-after-import) below (takes ~2 minutes)
6. Rename routes and stores on **Settings** to your real names
7. Enter go-live balances on **Opening_Balances**
8. Print route forms from [`templates/route-form-th-en.html`](../templates/route-form-th-en.html)
9. Delete sample rows in `Route_Sales`, `Store_Payments`, and `Expenses` before go-live

The workbook includes a **README_Setup** tab with the same dropdown instructions.

## Tabs

| Tab | Who edits | Purpose |
|-----|-----------|---------|
| Settings | Owner | Route names, store names, expense categories |
| Route_Sales | Bookkeeper | One row per store visit (cash or credit) |
| Store_Payments | Bookkeeper | When a store pays their debt |
| Store_Balances | Nobody (formulas) | Who owes what |
| Expenses | Bookkeeper | All money out |
| Inventory | Bookkeeper | Weekly bags produced vs sold |
| Opening_Balances | Owner (once) | Cash, bank, per-store debts on go-live |
| Dashboard | Nobody (formulas) | Monthly P&L |
| Cash_Flow | Nobody (formulas) | Monthly cash vs bank |
| README_Setup | Nobody | Dropdown fix instructions after Google import |

## Roles

### Driver (no Google Sheets access)

- Uses printed [`route-form-th-en.html`](../templates/route-form-th-en.html) only
- Fills one row per store: name, bags, Cash or Credit, amount
- Returns form to main branch — no internet needed on route

### Bookkeeper

- Enters paper form data into **Route_Sales** (one row per store)
- Enters **Expenses** and **Store_Payments** when stores pay
- Checks **Store_Balances** for who still owes money
- Adds weekly **Inventory** row; reviews **Dashboard** and **Cash_Flow** monthly

### Owner

- Maintains **Settings** (route and store names)
- Sets **Opening_Balances** on go-live
- Protects formula tabs (Store_Balances, Dashboard, Cash_Flow)

## Daily workflow

### Driver (no internet on route)

1. Fill one **paper route form** per route per day
2. One line per store: store name, bags, Cash or Credit, amount
3. Return form to main branch

### Bookkeeper

1. Enter each store line into **Route_Sales**
2. Enter expenses into **Expenses** as they happen
3. When a store pays → **Store_Payments**
4. Check **Store_Balances** for outstanding debts

### Weekly

1. Add one **Inventory** row (bags produced; sold is auto-summed from Route_Sales)

### Monthly

1. Set report month (`YYYY-MM`) on **Dashboard** and **Cash_Flow** cell B3
2. Review P&L and reconcile expected cash/bank to reality

## Store balance formula

For each store:

```
Still owed = Opening debt + Credit sales − Payments received
```

- **Opening debt:** `Opening_Balances` per-store rows
- **Credit sales:** SUM of `Route_Sales` where Sale type = Credit
- **Payments:** SUM of `Store_Payments` for that store

## Protecting formula tabs

In Google Sheets:

1. **Store_Balances**, **Dashboard**, **Cash_Flow** → select all → **Data → Protect sheets**
2. Leave **Route_Sales**, **Store_Payments**, **Expenses**, **Inventory**, **Opening_Balances** editable for bookkeeper

## Adding routes and stores

1. **Settings** tab → add name in next blank row under Routes (column A) or Stores (column C)
2. Dropdowns on input tabs pick up new names automatically — they read from named ranges (`Routes`, `Stores`) that point at those columns

## Dropdown setup after import

Google Sheets often drops or breaks Excel data validation on `.xlsx` import. The template is built to minimize this:

- **Named ranges** on the workbook (`Routes`, `Stores`, `ExpenseCategories`, `SaleTypes`, `PaymentMethods`) point at lists on **Settings**
- **One rule per column** (not thousands of per-cell rules)

### Check if dropdowns work

1. Open **Route_Sales** → click cell **B2** (Route column)
2. You should see a **dropdown arrow** in the cell
3. Repeat for **C2** (Store) and **E2** (Sale type)

### If dropdowns are missing — fix in place (keep your data)

You do **not** need to re-import if you already entered data. Re-apply validation on the live sheet:

**Option A — Manual (no code)**

For each column below: select the range → **Data → Data validation** → **Dropdown from a range** → enter the named range (no `=` needed in the UI; Google adds it).

| Tab | Column | Range to select | Named range |
|-----|--------|-----------------|-------------|
| Route_Sales | B | B2:B501 | `Routes` |
| Route_Sales | C | C2:C501 | `Stores` |
| Route_Sales | E | E2:E501 | `SaleTypes` |
| Store_Payments | B | B2:B501 | `Stores` |
| Store_Payments | D | D2:D501 | `PaymentMethods` |
| Expenses | B | B2:B501 | `ExpenseCategories` |
| Expenses | D | D2:D501 | `PaymentMethods` |
| Opening_Balances | A | A8:A107 | `Stores` |

Verify named ranges exist: **Data → Named ranges** — you should see `Routes`, `Stores`, etc. If missing, create them pointing at the Settings ranges in the table above (e.g. `Routes` → `Settings!A2:A100`).

**Option B — Apps Script (fastest)**

1. **Extensions → Apps Script**
2. Paste contents of [`scripts/google-sheets-validation.gs`](../scripts/google-sheets-validation.gs)
3. **Run → `applyLedgerValidations`** (authorize when prompted)
4. All input columns get dropdowns in one step

### If starting fresh — re-import

If you have no data yet, re-import the latest `ice-business-ledger.xlsx` with **Replace spreadsheet**, then test dropdowns again. Run Option A or B only if import still drops them.

## Google Sheets formula notes

After import, verify:

- Date columns are formatted as dates
- Amount columns use custom number format: `฿#,##0`
- `EDATE` and `SUMIFS` work in Google Sheets (they do)
- Named ranges survived import (**Data → Named ranges**)

### Why the old template broke dropdowns

The first version of the workbook had two problems:

1. **No named ranges** — each dropdown used a raw cross-sheet formula like `=Settings!$A$2:$A$100`. Google Sheets often ignores these when converting Excel validation rules.
2. **Per-cell rules** — Route_Sales alone had 1,500 separate validation objects (one per row × three columns). Importers frequently drop or corrupt bulk rules like that.

The current template uses five workbook named ranges and eight column-level rules total. If Google Sheets still strips them on import, use the [fix-in-place steps](#dropdown-setup-after-import) above — you keep all formulas and sample data.
